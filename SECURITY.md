# Security Policy

## Supported versions

AtlasX is experimental. Security fixes are expected on the latest main branch
until formal releases are published.

## Reporting a vulnerability

Please open a private security advisory or contact the maintainers. Do not post
secrets, vulnerable documents, private PDFs, or proprietary research data in a
public issue.

## Data handling guidance

- Never commit `.env` files or API keys.
- Never commit copyrighted PDFs or licensed full text.
- Use local ignored folders for private inputs.
- Review generated outputs before sharing them publicly.
- Remove sensitive notes, author correspondence, or institutional data before
  filing issues.
